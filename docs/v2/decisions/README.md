# V2 Decisions

This directory is reserved for ADR-style records.

Use it for:

- major sequencing choices
- architectural boundaries
- adoption or rejection of important trials
- policy decisions that need a stable "why"

Do not use it for:

- generic design prose
- raw experiment output
- brainstorming notes

Suggested ADR shape:

1. Title
2. Date
3. Status
4. Context
5. Decision
6. Consequences
7. Related artifacts

Examples of the kinds of choices that belong here:

- registry before broader automation
- shortlist-only VLM reranking
- official-source-first enrichment posture
- hook candidate as an explicit artifact family
